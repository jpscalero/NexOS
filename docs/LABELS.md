# Estándar de Etiquetas (Labels) de NexOS

Para mantener el proyecto organizado, utilizaremos el siguiente esquema de etiquetas en GitHub:

## 🔴 Prioridad y Gravedad
- `priority:critical`: Errores que impiden la construcción de la ISO o comprometen la seguridad.
- `priority:high`: Errores importantes o características clave.
- `priority:low`: Mejoras menores o sugerencias estéticas.

## 🔵 Tipos de Tarea
- `type:bug`: Algo no funciona como se esperaba.
- `type:feature`: Nueva funcionalidad o herramienta.
- `type:enhancement`: Mejora de algo ya existente.
- `type:documentation`: Cambios en la Wiki o README.

## 🟢 Categorías del Sistema
- `area:core`: Relacionado con los scripts de construcción y base Debian.
- `area:pentest`: Relacionado con las herramientas de seguridad.
- `area:ui`: Relacionado con KDE, SDDM o estética.
- `area:server`: Relacionado con Nginx, MariaDB, SSH.

## 🟡 Estado de Colaboración
- `good first issue`: Ideal para nuevos colaboradores.
- `help wanted`: Se busca ayuda específica de la comunidad.
- `in-progress`: Tarea siendo trabajada actualmente.
